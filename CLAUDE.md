# medialab-stack

이 프로젝트는 미디어아트 창작·연구·교육을 위한 Claude Code 스킬팩이다.

## 스킬 명령어

- `/imagine` — 컨셉 발상 파트너. CONCEPT.md 생성
- `/sketch` — 즉시 프로토타이핑. 최소 동작 코드 생성
- `/vary` — 기존 스케치의 변형 생성
- `/experiment` — 파라미터 공간 탐색과 체계적 실험
- `/glitch` — 규칙을 깨뜨려 우연의 미학 발견
- `/survey` — 레퍼런스 리서치 (선행 작품, 기술, 영감)
- `/digest` — 논문/기술 문서를 미디어아트 관점으로 요약
- `/cite` — REFERENCES.md를 학술 인용 형식으로 변환
- `/critique` — 형식·개념·관객 경험 관점의 미학적 리뷰
- `/optimize` — 퍼포먼스 최적화 (미학적 영향 설명 포함)
- `/install` — 전시 배포와 자동시작 스크립트
- `/document` — 아티스트 스테이트먼트, 기술 문서, 전시 캡션
- `/archive` — 프로젝트 정리와 카탈로그 생성
- `/explain` — 코드에 교육적 주석을 달아 해설 버전 생성
- `/workshop` — 완전한 워크숍 자료 세트 생성
- `/exercise` — 3단계 과제 생성 (기초/응용/창작)
- `/syllabus` — 주차별 커리큘럼 설계

## 메타 문서

프로젝트 루트에 다음 문서들이 생성·관리된다. 모든 스킬은 실행 시 이 문서들의 존재를 확인하고, 있으면 자동으로 읽어 컨텍스트로 사용한다.

- **CONCEPT.md** — 작품의 예술적 의도, 컨셉, 기술 스택
- **REFERENCES.md** — 레퍼런스 누적 (선행 작품, 기술, 영감)
- **CATALOG.md** — 프로젝트 파일 카탈로그
- **TECHNICAL-RIDER.md** — 전시 기술 사양서
- **EXPERIMENT-LOG.md** — 실험 기록

## 행동 원칙

1. **"실행 가능한 실험" 우선** — "완벽한 코드"보다 빠르게 돌아가는 최소 실험을 만든다.
2. **탐색 우선** — 기술적 정확성과 예술적 탐색 사이에서 탐색을 우선한다.
3. **미학적 판단 존중** — 사용자의 미학적 선택을 존중한다. AI가 "더 나은" 것을 강요하지 않는다.
4. **비파괴적** — 사용자의 기존 파일을 직접 수정하지 않는다. 새 파일 생성이 기본.
5. **한 번에 하나의 질문** — 사용자에게 여러 질문을 동시에 하지 않는다.

## 프레임워크 감지

프로젝트 파일 구조를 보고 크리에이티브 코딩 프레임워크를 자동 감지한다:

- `package.json`에 p5 → p5.js
- `.pde` 파일 → Processing
- `ofApp.h`/`ofApp.cpp` → openFrameworks
- `.toe` 파일 → TouchDesigner
- `.maxpat` 파일 → Max/MSP
- 감지 불가 시 사용자에게 질문
